# Jiang v0.2.0

`v0.2.0` 是 Jiang 的第一个正式 Stage2 bootstrap release。

## Highlights

- Stage2 (`compiler/`) 进入唯一继续演进的编译器主线
- `script/build_stage2.sh` 默认优先使用已有 `stage2c` 自举重编
- `script/stage2_selfhost_smoke.sh` 已固定验证：
  `stage2c -> compiler/entries/compiler.jiang -> stage2c.selfhost -> stage2c.roundtrip`
- Stage2 当前已覆盖：
  - `frontend -> HIR -> JIR -> C/LLVM`
  - 多模块与 `public` / alias import
  - `struct` / `union` / `enum`
  - `UInt8` / `UInt8[]` / array / pointer
  - `emit-c` / `run` / `selfhost` / `error` / `llvm` / `llvm-error`
- `union` 现已支持隐式 tag：

```jiang
union Result {
    Int ok;
    Int err;
}
```

## Bootstrap Meaning

从这个版本开始：

- `v0.2.0` 可以作为后续 Stage2 主线的 bootstrap compiler
- 后续开发应优先使用“上一版正式 release 包中的 `jiang`”
- `stage1c` 降级为冷启动/灾备兜底路径

## Validation

发布前通过：

- `bash ./script/build_stage2.sh`
- `bash ./script/stage2_selfhost_smoke.sh`
- `bash ./script/stage2_complete_smoke.sh`

## Upgrade Guidance

推荐把 release 提供的 `jiang` 安装到固定位置，例如：

```bash
~/.jiang/toolchains/v0.2.0/bin/jiang
```

然后在构建主线 Stage2 时设置：

```bash
export STAGE2_BOOTSTRAP_STAGE2=$HOME/.jiang/toolchains/v0.2.0/bin/jiang
bash ./script/build_stage2.sh
```

仓库内部开发产物仍保持 `build/stage2c`，但正式 release 对外二进制名称固定为 `jiang`。
