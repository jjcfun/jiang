# Jiang Programming Language

Jiang 是一门旨在成为编程领域“银弹”的现代静态类型编程语言。它的核心设计哲学是 `All in one`，结合了系统级语言的底层内存控制与现代高级语言的丰富表达能力，追求在性能、安全性与开发体验之间达到完美的平衡。

## 🌟 核心特性 (Key Features)

*   **极简而统一的类型系统**：遵循“从左往右，从里到外”的类型推导与声明逻辑（如 `Int?[2][3]`）。
*   **内存安全与所有权机制**：内置轻量级的所有权（Ownership）校验，结合手动的内存分配与释放（`new` 与 `sudo $free()`），在无 GC 的前提下保障内存安全。
*   **透明的智能指针（自动解引用）**：使用指针如字面量一般自然，而底层的代理与操作则隐藏在 `$` 语法之下（如 `$ptr.move()`）。
*   **高度灵活的聚合类型**：支持强大且直观的结构体（Struct）、多态枚举（Enum）、以及安全的联合体（Union）与模式匹配。
*   **泛型及无缝异步设计**：内置函数与结构体级别的泛型修饰，以及为适应现代 IO 设计的 `async` 原生支持。

## 🏗 架构演进与编译器实现路线图

Jiang 语言的落地计划分为**两个核心阶段**。我们将采用循序渐进的策略：先用成熟的底层语言建立滩头阵地，随后借助工业级的优化框架实现语言自身的“硬自举”。

### 阶段一：原型落地（C Transpiler 软自举雏形）
**目标**：“先让语言跑起来，快速验证语义设计”。
在没有可用编译器前，我们采用 C 语言作为宿主，手写一个 `Jiang -> C` 的前端转译器。它负责基础的词法、语法解析与语义检查，然后交由 GCC/Clang 完成底层的系统级翻译与平台分发。

> **当前状态**：🚧 **规划中 / 早期开发中**

### 阶段二：工业级自举（LLVM IR 硬核化）
**目标**：“使用 Jiang 编写 Jiang 编译器，对接 LLVM 成为独立的高性能生态”。
当第一阶段的 C 版本编译器稳定后，我们将使用 Jiang 语言本身重写编译器前端（Lexer & Parser），并将后端的代码生成器直接对接给 **LLVM 中间表示层（LLVM IR）**。此时，Jiang 将彻底摆脱生成 C 代码的桎梏，独立生成极度优化的原生机器码或 WebAssembly。

## 🎯 里程碑规划 (Milestones)

### Phase 1: The C-Transpiler Base (C 语言转译器实现)

*   [ ] **M1.1: AST 与词法/语法分析引擎 (Lexer & Parser)**
    *   定义核心 Token 结构与 AST 节点结构。
    *   实现支持基础类型、变量声明、控制流的解析（输出标准格式化 AST）。
*   [ ] **M1.2: 语义分析与符号表 (Semantic Checking)**
    *   实现作用域隔离。
    *   实现基础的类型推导与强静态类型校验。
*   [ ] **M1.3: 所有权与内存安全检查层 (Borrow Checker MVP)**
    *   扫描 AST，对 `move` 语义和指针赋值进行所有权状态跟踪。
    *   实现在编译期拦截“使用移出变量（Use-After-Move）”的致命内存错误。
*   [ ] **M1.4: C 代码生成器 (C-CodeGen)**
    *   遍历 AST，映射为包含标准 C 库代码的源文件结构。
    *   处理泛型的单态化（Monomorphization），将 Jiang 的抽象函数展开在 C 层。
*   [ ] **M1.5: 第一阶段自举闭环 (The "Soft" Bootstrap)**
    *   构建基本标准库（Minimal Stdlib: print, file, string）。
    *   使用这个初级 C 编译器，成功编译一个用 Jiang 语言写成的简单词法分析程序。

### Phase 2: The LLVM Evolution (基于 LLVM 的自举进化)

*   [ ] **M2.1: Jiang 语言重写前端引擎**
    *   正式使用 Jiang 语言（由 M1 编译）完整重写第一阶段的 Lexer。
    *   由于拥有强大的模式匹配与枚举（Union），重写过程将极大压缩代码量并增加鲁棒性。
*   [ ] **M2.2: 重构编译器内部结构：MIR 数据流化**
    *   在编译器内部建立 `HIR (High-level IR)` (用于类型与所有权推导) 与 `MIR (Mid-level IR)` (用于为 LLVM 生成奠定规范的基础)。
*   [ ] **M2.3: 对接 LLVM 基础设施层**
    *   引入 LLVM 相关的库绑定 (如 LLVM-C API / C++ API)。
    *   编写针对 LLVM 的数据格式绑定模块，将 MIR 数据流与 LLVM 的 Basic Block 对齐。
*   [ ] **M2.4: LLVM 代码生成器核心实现 (IR Emitter)**
    *   针对 Jiang 的基本类型生成对于的 LLVM DataLayout。
    *   精准翻译函数调用、泛型映射、隐式解引用行为为 LLVM Instrinsic / IR 指令集。
*   [ ] **M2.5: Wasm 脱潜与多平台本地化分发**
    *   彻底脱离 C 编译器依赖。底层直接调配 `llvm-target` 工具链。
    *   跑通 `jiangc main.qing -target wasm32` 乃至 `x86_64`、`aarch64` 等原生交叉编译命令，借助 LLVM 的 O3 优化产出无敌性能。
